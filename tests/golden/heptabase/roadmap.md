# 课程路线图

当你拿到这份包含 56 个细分章节、总字数超过 52 万字的文档时，第一反应可能是：“我真的能学完吗？”这种压力很正常。对于一个旨在让你从嵌入式/硬件工程师转型为边缘 AI 开发者的项目，广度（从硬件电路到 PyTorch 训练）和深度（从模型量化到 C++ 并发框架）确实惊人。但请记住：这门课的设计初衷并不是让你死记硬背，而是为你提供一个可以带走的工程模板。
就像要把大象装进冰箱，我们需要分三步：打开门、放进去、关上门。本节就是带你“打开门”，看清这头大象的全貌，以及我们要走过的七个模块。

## 模块化学习：七大阶段构成的工业 AI 闭环
这门课程被划分为七个逻辑严密的模块，每一个模块都对应工业 AI 项目中不可或缺的一环。这种划分方式不是为了增加负担，而是为了让你在每一个阶段都能看到明确的产出。 [1]

### 1. 项目总览与全局认知
在写任何代码之前，我们先要解决“为什么”的问题。在这个模块，你将了解到 P1 火焰检测和 P2 轴承故障诊断这两个子项目的背景。你会发现，它们不是简单的 Demo，而是基于企业真实独家数据集开发的。 [2] 我们会分析工业数据的噪声复杂、类别不均衡等真实痛点，并向你展示整个系统的“五层架构”是如何构思出来的。

### 2. 硬件原理与信号链
作为嵌入式工程师，不理解硬件就无法控制系统。我们会深入剖析继电器驱动、LED 指示、ADXL1003 三轴振动采集模块以及 24-bit 高精度 ADC 的信号链设计。这一阶段的目标是让你明白，数据是如何从传感器那一端的物理信号，一步步转化为 Linux 操作系统可以处理的数字流的。

### 3. PC 端训练与算法创新 (MSFD)
这是算法的核心模块。我们将离开树莓派，回到算力充沛的 PC 端。
你将学习到创新点 1：MSFD 多信号融合算法。
你将深度参与创新点 2：VariableMobileNet 自研模型的训练过程。 [3]
我们将从数据预处理、模型架构设计、训练循环到最终的 GradCAM 可解释性分析，走过一条完整的算法开发路径。

### 4. 模型优化：量化与压缩
为什么一定要有这一章？因为在 PC 上跑得顺的模型，直接丢到树莓派上可能会卡成幻灯片。
这里有我们的创新点 3：基于代表性样本筛选的 INT8 静态量化策略。 [4] 你将学习如何用 K-Means 聚类和预测熵来选取校准集，从而在不损精度的情况下，最大程度地压榨硬件性能。

### 5. 树莓派部署与工程化
这是从 Python 转到 C++ 的关键跳板。我们将配置树莓派的 Linux 环境，安装依赖库，并学习如何使用 CMake 进行条件编译。仿真模式（--simulate）将是你最得力的助手，它让你在没有硬件传感器的情况下也能调试整条 AI 推理链路。

### 6. Linux C++ 五层统一框架 (核心中的核心)
这是全课字数最多、含金量最高的部分。我们将逐文件解析这套“可复用、可扩展、可迁移”的五层框架。你将看到：
如何用多线程解耦采集、处理和推理。
如何利用单例模式管理配置，用 RAII 管理并发锁。
为什么要把代码分成 DataAcquisition、AIModel、ExtendedFeatures 等层级。
掌握这套框架，你带走的就不只是一个项目，而是一套工业级代码设计哲学。 [5]

### 7. 面试问答与简历收尾
所有的技术积累，最终都要转化成面试时的侃侃而谈。我们会整理高频面试问答题库，并教你如何将 MSFD 算法、VariableMobileNet、INT8 量化这些亮点巧妙地写进简历，支撑起你求职嵌入式 AI 岗位的竞争力。

## 你的三大核心产出
学完这门课，你手里不仅有几百页的笔记，还有三样可以实实在在改变你职业路径的东西：
一套自研轻量化模型代码：基于 VariableMobileNet 的训练、量化、评估全流程脚本。它不是通用的 ResNet 或 MobileNet，而是针对边缘计算场景大幅精简后的创新版本，能证明你具备模型优化能力。
一套 Linux C++ 五层应用框架：这是一个工业级的系统框架，涵盖了并发队列、多外设驱动、AI 推理引擎封装。你可以把它作为基础，快速开发出任何基于 Linux 的 AI 应用。
一份极具竞争力的简历项目：相比于简历上泛滥的“音视频播放器”或“智能家居监控”，这套基于真实工业场景、包含核心算法创新的 AI 闭环项目，更能让面试官眼前一亮。 [6]

## 学习策略：先整体后细节，按需补齐 C++
针对初学者，这门课采用了两套关键的学习策略，旨在帮你平稳度过“知识焦虑期”。

### 1. “大图景”优先策略
在本课程中，每一章、甚至每一个重要文件开始前，我们都会先给出一个 Big Picture（全局视野）。
为什么要这样设计？ 比如在学 C++ 五层框架时，如果直接开始看具体的 .cpp 实现，你很快就会迷失在复杂的逻辑中。
我们的做法：先讲这一层在整个数据流中的位置，讲它的输入输出是什么，讲为什么要这样设计。等你心里有了“全局地图”，再去抠代码细节，就会有一种“原来如此”的成就感。

### 2. “穿插式” C++ 学习法
很多同学担心自己 C++ 基础差。请注意：我们不会让你单独花一个月去啃厚厚的 C++ 语法书。
按需补齐：所有的核心语法（如 std::unique_ptr 智能指针、std::mutex 互斥量、模板类、原子操作等）都会在对应的工程位置出现。
场景记忆：当我们在第 5 章遇到需要保护线程安全的数据队列时，我们会顺便讲解 std::condition_variable。这种结合工程场景的学习方式，比单纯看书效率高出 10 倍以上，且学完就能在项目里用起来。

## 给你的建议：如何面对 52 万字文档
如果你在学习过程中感到疲惫，请记住以下三点：
原子化章节设计：我们将 7 大模块拆分成了 56 个微小的 Topic。这种“原子化”设计是为了让你利用碎片时间也能推进进度。不要想一次性吞掉整个模块，每次只攻克一个 Topic 即可。
不要迷失在文档中：文档详细是为了方便你查阅细节，而不是要求你逐字背诵。对于已经掌握或者偏原理的内容，可以快速浏览；对于带代码解析的章节，则需要你动手跑一跑。
心态建设：本项目的深度足以支撑起一份高薪职位。你现在感到的每一个难点，都是你未来在面试场上甩开竞争对手的护城河。
这套路线图是你未来的指南针。当你觉得代码逻辑太复杂或者理论太枯燥时，回来看看这张图，确认自己所在的方位。现在，让我们正式踏入第一章：认识我们要解决的两个真实工业场景。
